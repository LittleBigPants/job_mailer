## Tech Debt Details

### 1. outreach_log.csv not gitignored

**File:** `.gitignore`
**Issue:** `outreach_log.csv` is created at runtime in the working directory and will contain email addresses and generated content. The file will be tracked by git on first run.
**Fix:** Add `outreach_log.csv` to `.gitignore`.
**Affects:** LOG-01, LOG-03 (operational, not functional)

### 2. Status.SKIPPED is dead code

**File:** `src/job_mailer/models.py` line 18
**Issue:** `Status.SKIPPED = "skipped"` is defined but never set by any code path. The idempotency branch silently `continue`s without setting record status or calling `log_record`. Skipped URLs have no log entry.
**Fix:** Either remove `Status.SKIPPED` (if silent skipping is intended) or set it and call `log_record` in the idempotency branch (if skip events should be logged for observability).
**Affects:** LOG-03 (skip functionality works; skip observability is absent)

### 3. tenacity declared but unused

**File:** `pyproject.toml` line 14
**Issue:** `tenacity>=9.1.4` is a runtime dependency but is imported nowhere. The generator's retry logic is implemented manually.
**Fix:** Remove `tenacity` from `[project.dependencies]`.
**Affects:** None (no functional impact; adds unnecessary install overhead)

### 4. README incorrectly documents CSV column passthrough

**File:** `README.md` line 137
**Issue:** "Additional columns (company name, notes) are passed through to the prompt template" is false. `__main__.py` reads only `row[0]`.
**Fix:** Update README line 137 to match `--input` help text: "Single-column CSV file with one URL per row."
**Affects:** INPUT-01 (documentation accuracy only; code is correct)

### 5. scraper.py uses print() for warnings

**File:** `src/job_mailer/scraper.py` line 134
**Issue:** `print()` bypasses typer's output capture, making per-page fetch warnings invisible in `CliRunner`-based tests.
**Fix:** Change to `typer.echo(..., err=True)` for consistency.
**Affects:** Minor consistency only; no requirement is broken
