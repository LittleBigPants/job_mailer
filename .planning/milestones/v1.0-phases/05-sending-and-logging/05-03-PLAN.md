Current __main__.py structure (src/job_mailer/__main__.py):
```python
from job_mailer.config import check_env, load_profile, validate_profile
from job_mailer.generator import generate_email
from job_mailer.scraper import scrape_company

@app.command()
def main(input: Path = typer.Option(..., "--input", ...)) -> None:
    check_env()
    profile = load_profile()
    validate_profile(profile)
    typer.echo(f"Config loaded. Processing: {input}")
    with open(input, newline="") as fh:
        reader = csv.reader(fh)
        for row in reader:
            if not row or not row[0].strip(): continue
            url = row[0].strip()
            if url.lower() == "url": continue
            try:
                record = scrape_company(url)
                if record.email_found:
                    record = generate_email(record, profile)
                if record.generated_message:
                    wc = len(record.generated_message.split())
                    typer.echo(f"  {record.company_name} — {record.email_found} — {wc} words — {record.status.value}")
                else:
                    typer.echo(f"  {record.company_name} — {record.email_found or 'no email'} — {record.status.value}")
            except Exception as exc:
                typer.echo(f"  WARNING: failed to scrape {url}: {exc}", err=True)
    typer.echo("Done.")
```

    1. Before the loop, initialize counters and read delay:
       - sent_count = 0; failed_count = 0; no_email_count = 0
       - delay = profile.get("send", {}).get("delay_seconds", 2)

    2. Inside the loop, after scrape + generate (preserve existing scrape/generate calls):
       - If record.generated_message is set: call send_email(record, profile) inside try/except
         - try: record = send_email(record, profile)
         - except resend.exceptions.RateLimitError (daily_quota_exceeded path):
             - record.status = Status.RATE_LIMITED (already set by send_email before re-raise)
             - log_record(record)
             - typer.echo(f"ERROR: Resend daily quota exceeded. Sent {sent_count} emails this run. Remaining companies not processed.", err=True)
             - sys.exit(1)
         - After send_email returns: log_record(record)
         - Echo based on record.status:
           - Status.SENT: typer.echo(f"  {record.company_name} — {record.email_found} — sent"); sent_count += 1
           - Status.RATE_LIMITED: typer.echo(f"  WARNING: {record.company_name} — rate_limited (429)", err=True); failed_count += 1
           - else (SEND_FAILED): failed_count += 1 (no echo for generic failures)
       - If record.generated_message is NOT set (no email found or generation failed):
           - no_email_count += 1 only when record.status in (Status.NO_EMAIL_FOUND, Status.SCRAPE_FAILED)
           - failed_count += 1 when record.status == Status.GENERATION_FAILED
           - log_record(record)  # log all outcomes, not just sends
       - time.sleep(delay) at the very end of each iteration (after logging)

    3. After the loop (outside it): typer.echo(f"Done. {sent_count} sent, {failed_count} failed, {no_email_count} no email found.")

    Remove the old "wc = len(...)" echo lines — they are replaced by the new status-aware echo.

    Keep the outer try/except Exception that catches scrape/unexpected errors — it should also call log_record if a record was created before the exception, but if the exception is before scrape_company returns any record, just echo the warning (existing behavior).
